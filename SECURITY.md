# Security Policy

## Reporting a vulnerability

Please **do not open a public issue** for security problems.

Email **security@buildingsync.app** with:
- Description of the issue
- Steps to reproduce
- Affected URL or component
- Your name / handle for credit (optional)

We aim to acknowledge within 72 hours and ship a fix or mitigation within 14 days for high-severity issues.

## Scope

In scope: `buildingsync.app`, `admin.buildingsync.app`, code in this repository.

Out of scope: rate-limiting / brute-force without proof of impact, missing security headers without an exploit path, social engineering, physical attacks.

## Safe harbour

We will not pursue legal action against good-faith research that:
- Avoids data destruction, exfiltration of others' data, or service disruption
- Stops at proof-of-concept and reports promptly
- Does not publicly disclose before a fix ships
